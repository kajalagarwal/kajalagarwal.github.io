Difference between Class and id in CSS.

Class

  • Classes are not unique

    1. You can use the same class on multiple elements.
    2. You can use multiple classes on the same element.
    3. Combining class selectors

      4. Any styling information that needs to be applied to multiple objects on a page should be done with a class. Take for example a page with multiple "div" but same class "wedge" is allowed.

      You can use multiple classes to style an HTML element but you can only use one ID when styling an HTML element. This means that class selectors have a wider range of applications

Id's

  • Id's are unique

    1. Each page can have only one element with that ID
    2. Each element can have only one ID
    3. IDs have higher specificity than classes

      4. In id's you need to understand that your code will become invalid if you applied same id to more then 1 element which is not the case in classes. This is why mostly programmers prefer to use class over id because it reduce redundancy.

      There may be times when a declaration conflicts with another declaration. These conflicts are resolved using the Cascade rules. In simple terms, if a class selector and ID selector were to be in conflict, the ID selector would be chosen.
