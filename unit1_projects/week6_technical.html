Ruby Concepts


Ruby is an awesome piece of creation, Everyday I'm learning and discovering ruby in lot different and better way. just like other days today I dug more into how ruby closures work. here is my brief explanation after reading about ruby closures. thanks to the author for his wonderful writeup.
In my Ii have worked in many different programming languages and ruby always stay one step apart from them because of its easiness and inbuilt function.
As we know ruby does lotta stuffs with closure, like in array iteration we are using closures in everywhere. Ruby has 4 type of closures - Block, Procs, Lambda and Method. I'm trying to explain them in brief. So i could use this post as my future reminder

Blocks

Block is a imprtant part of Ruby which delete the redundancy. It's used with '&' sign prefixed variable also executes using yield or *.call.

Block can be initialized as

=begin
Ruby Code blocks are chunks of code between braces or
between do- end that you can associate with method invocations
=end
def call_block
puts 'Start of method'
# you can call the block using the yield keyword
yield
yield
puts 'End of method'
end
# Code blocks may appear only in the source adjacent to a method call
call_block {puts 'In the block'}

The ouput of the above exemple is

Start of method
In the block
In the block
End of method

Procs

Its used as other ruby data types (array, hash, string, fixnum etc..), it's reusable and could be used across multiple calls. also several procedures could be passed at a time. I works mostly as Block so it is upto user which he found more better to understand.

Example of how method can take proc.

=begin
You cannot pass methods into other methods (but you can pass procs into methods),
and methods cannot return other methods (but they can return procs)
=end
def some_mtd some_proc
puts 'Start calling'
some_proc.call
puts 'End calling'
end
say = lambda do
puts 'Hello Proc'
end
some_mtd say

The output of the above example is


Start calling
Hello Proc
End calling

Lambda

Lambda is just the functon without the name. Lambdas in ruby are also objects which sneak into the code easily. Unlike proc the last value of lambda is its return value. Lambdas have methods that can be assigned to values.

Example Code

l = lambda do |string|
if string == "try"
return "There's no such thing"
else
return "Do or do not."
end
end
puts l.call("try")

The output of the abovecode is

There is no such thing
